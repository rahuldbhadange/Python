Integrator API¶

This API is in 2 parts

  1. The API on the Integrator API Object that the integrator calls
  2. The Integrator Callback Interface that the integrator has to implement

Integrator API reference¶

Integrator¶

class ioticlabs.dt.api.integrator.Integrator(config, client, callbacks)¶

Provides asset notifications as well as event publishing functionality.

assets¶

Sequence of assets (snapshot) known to integrator internally. on_asset_* callbacks should be used to determine what assets exist rather than polling this.

publish_event(event, confirm=True)¶

Publish an event for a given asset.

event - Instance of AssetEvent implementing class. If time is not specified, current UTC time will be used. The
source field is also ignored (and set based on configuration).
confirm - Confirm delivery of the event. If set, this method will block until the event has been published.
Otherwise it will return immediatley and publish failures will be ignored.
Raises:

TypeError - If the event is invalid AssetUnknown - If the integrator is currently not handling this asset EventPublishFailure - If confirm=True and the event could not be published. Note: Timeouts are goverend by

agent and integrator configuration.

UnknownEvent - event is not registered UnknownVersion - event version not known for registered event EncodeDecodeError - extra data cannot be encoded ValidationError - additional type data validation failed

start()¶

Starts the integrator. Other public methods herein must not be called beforehand. Should be called after starting Iotic agent. Alternatively use with keyword on the instance.

stop(timeout=10)¶

Stops the integrator. Should be called before shutting down the Iotic agent.

t2_respond(request, mime, data, wait=False)¶

Send response for the given type2 request. If a large amount of data is to be returned, t2_respond_streamed is preferred.

request - T2Request instance mime - Mime type of data (string, required) data - bytes (i.e. raw response) wait - whether to block until all underlying agent requests complete. Otherwise returns as soon as all requests

have been enqueued.
Raises:
TypeError - If the request is invalid ValueError - If mime or data are invalid AssetUnknown - If the asset is no longer known. In this case the original t2 request should be ignored
t2_respond_error(request, reason, wait=False)¶

Send failure response for the given type2 request.

request - T2Request instance reason - T2ProviderFailureReason instance wait - whether to block until underlying agent request completes. Otherwise returns as soon as the request has

been enqueued.
Raises:
TypeError - If the request or reason is invalid AssetUnknown - If the asset is no longer known. In this case the original t2 request should be ignored
t2_respond_streamed(request, mime, wait=False)¶

Streaming version of t2_respond. Provides context-managed function which accepts one argument: data (bytes). See t2_respond for more details. Expected usage:

with t2_respond_streamed(req, ‘application/some-mime’) as write:
for chunk in response_data:
write(chunk)

Integrator Callbacks reference¶

IntegratorCallbacks¶

class ioticlabs.dt.api.integrator.IntegratorCallbacks¶

Set of callbacks which integrator implementations must provide.

NOTE: Methods herein should not perform long-running tasks and instead delegate to make necessary changes.

on_asset_created(asset_id)¶

A new asset has been created. Called once for each known asset on startup as well as whenever a new asset appears whilst the integrator is running.

on_asset_deleted(asset_id)¶

An asset has been deleted. Called whenever an asset has been removed and should no longer be considered by the integrator. Note: This is NOT called if an asset has been deleted whilst the integrator is not running.

on_t2_request(request)¶

A new type2 request has been made for a particular asset.

request - instance of T2Request
