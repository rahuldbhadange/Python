Follower API¶

This API is in 2 parts

  1. The API on the Follower API Object that the follower calls
  2. The Follower Callback Interface that the follower has to implement

Follower API reference¶

Follower¶

class ioticlabs.dt.api.follower.Follower(config, client, callbacks)¶

Provides asset and event notifications as well as event confirmation functionality.

ack_event(event)¶

Acknowledge that an event has been handled. Any events with a smaller offset are also implied to have been handled. Unknown assets events will be ignored.

asset_id - Id of the asset event - Instance of AssetEvent implementing class

Raises:
TypeError - If the event is invalid
assets¶

Sequence of assets (snapshot) known to follower internally. on_asset_* callbacks should be used to determine what assets exist rather than polling this.

start()¶

Starts the follower. Other public methods herein must not be called beforehand. Should be called after starting Iotic agent. Alternatively use with keyword on the instance.

stop(timeout=10)¶

Stops the follower. Should be called before shutting down the Iotic agent.

t2_request(asset_id, type_, data=None, timeout=10, chunk_timeout=10)¶

Makes a type 2 request for the given asset. Generates tuples of response mime type and raw data (bytes).

asset_id - asset to which request applies type_ - type of request (string) data - any additional arguments specific to the request (non-empty bytes or None) timeout - how long to wait (in seconds, fractional) before receive first response chunk chunk_timeout - how long to wait (in seconds, fractional) before next expected (but not first) chunk arrives

Raises:
AssetUnknown - if the asset is not known to this follower TypeError - if type is invalid ValueError - if data or timeout is invalid T2ResponseException - if request failed to due a provider-related issue

Follower Callbacks reference¶

FollowerCallbacks¶

class ioticlabs.dt.api.follower.FollowerCallbacks¶

Set of callbacks which follower implementations must provide.

NOTE: Methods herein should not perform long-running tasks and instead delegate to make necessary changes.

on_asset_created(asset_id)¶

A new asset has been created. Called once for each known asset on startup as well as whenever a new asset appears whilst the follower is running.

on_asset_deleted(asset_id)¶

An asset has been deleted. Called whenever an asset has been removed and should no longer be considered by the follower. Note: This is NOT called if an asset has been deleted whilst the follower is not running.

on_event(event)¶
A new event has been published for a particular asset. Does not include internal events apart from:
IntTwinEventInvalid IntFollowerEventInvalid

event - instance of AssetEvent

on_event_internal(event)¶

Override to handle internal events (see api.event.internal). This does NOT include internal events explicitly listed by on_event.

NamedEventMixin reference¶

NamedEventMixin¶

class ioticlabs.dt.api.follower.NamedEventMixin¶

Routes on_event calls to functions named do_<EVENT_TYPE_LOWERCASE> (ignoring events for which no function has been defined. Note: With this mixin included, on_event is overridden. If one is interested in handling (almost) all events the same way, this mixin might not be applicable.

E.g. for an event of type AssetEvent the method definition (can also be class/static) would be:
def on_assetevent(self, event)
on_event_unmatched(event)¶

Called for any events which do not have a named “on_*” function. Should at least acknowledge events one is not interested in.

Type 2 exceptions¶

Exceptions¶

class ioticlabs.dt.api.follower.T2Unavailable¶

Type2 functionality has not been enabled in this follower

class ioticlabs.dt.api.follower.T2Timeout¶

The response for a type 2 request has timed out

class ioticlabs.dt.api.follower.T2ResponseException(uref, reason)¶

Raised for error responses for a type 2 request.

Available attributes:

uref - request reference reason - One of T2ReqFailureReason

Reasons¶

class ioticlabs.dt.api.follower.T2ReqFailureReason¶

Failure reasons for type 2 requests (prior to data transfer)

PROVIDER_UNAVAILABLE = 2¶

Provider not reachable

PROVIDER_UNKNOWN = 1¶

No provider for request

REQ_UNHANDLED = 3¶

Provider cannot handle request

RESOURCE_UNKNOWN = 4¶

Provider cannot find information for this request, asset and parameters
