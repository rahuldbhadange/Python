NestedConfig API¶

This is the object that can be used to retrieve and validate config items quickly and easily.

NestedConfig API reference¶

class ioticlabs.dt.api.util.NestedConfig¶

For retrieving and checking nested configuration in Mappings

class Check¶

Common checks to apply when using get()

static non_empty_str(item)¶

Non-empty string

classmethod seq_non_empty_str(seq, zero_items_allowed=False)¶

Sequence of non-empty strings, optionally with no elements

static get(root, dot_path, default=None, required=True, check=None)¶

Retrieves item from nested mapping.

root - the mapping from which to retrieve dot_path - path to item, e.g. ‘a.b.c’ retrieves root[‘a’][‘b’][‘c’] required - Whether to raise exception if not found (rather than returning default) default - what default to return if item not found. check - either a function which validates found item, a type to check for or None for no checking

Returns found item or raises ValueError if not found or invalid.
